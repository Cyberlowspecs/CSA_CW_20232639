Smart Campus API - Test Execution Guide

This document contains the cURL commands adapted for our API implementation to execute the full testing order. 
Note: Our base API path is exactly `http://localhost:8080/smart-campus-api/api/v1`.

1. Discovery endpoint
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/
Expected: 200 OK (Returns JSON with metadata and _links)

2. Check rooms list at start
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/rooms
Expected: 200 OK (Returns empty list `[]`)

3. Create a room just for delete-success test
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/rooms \
  -H "Content-Type: application/json" \
  -d '{"id": "DEL-101", "name": "Delete Test Room", "capacity": 10}'
Expected: 201 Created

4. Get that room
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/rooms/DEL-101
Expected: 200 OK

5. Delete that room successfully
curl -i -X DELETE http://localhost:8080/smart-campus-api/api/v1/rooms/DEL-101
Expected: 204 No Content (Standard JAX-RS success status for DELETE)

6. Confirm deleted room is gone
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/rooms/DEL-101
Expected: 404 Not Found

7. Create the main room for all remaining tests
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/rooms \
  -H "Content-Type: application/json" \
  -d '{"id": "LIB-301", "name": "Library Quiet Study", "capacity": 80}'
Expected: 201 Created

8. Get main room by ID
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/rooms/LIB-301
Expected: 200 OK

9. Duplicate room check
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/rooms \
  -H "Content-Type: application/json" \
  -d '{"id": "LIB-301", "name": "Library Quiet Study", "capacity": 80}'
Expected: 201 Created (Note: Our basic HashMap implementation currently overwrites existing keys. To strictly enforce 409 Conflict, an explicit existence check would be added to the POST method.)

10. Invalid sensor roomId test
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors \
  -H "Content-Type: application/json" \
  -d '{"id": "CO2-999", "type": "CO2", "status": "ACTIVE", "currentValue": 420.0, "roomId": "NO-ROOM"}'
Expected: 422 Unprocessable Entity (Triggered by LinkedResourceNotFoundException)

11. Create valid CO2 sensor
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors \
  -H "Content-Type: application/json" \
  -d '{"id": "CO2-001", "type": "CO2", "status": "ACTIVE", "currentValue": 420.0, "roomId": "LIB-301"}'
Expected: 201 Created

12. Create valid temperature sensor
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors \
  -H "Content-Type: application/json" \
  -d '{"id": "TEMP-001", "type": "Temperature", "status": "ACTIVE", "currentValue": 22.5, "roomId": "LIB-301"}'
Expected: 201 Created

13. Get all sensors
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/sensors
Expected: 200 OK

14. Filter sensors by CO2
curl -i -X GET "http://localhost:8080/smart-campus-api/api/v1/sensors?type=CO2"
Expected: 200 OK

15. Filter sensors by Temperature
curl -i -X GET "http://localhost:8080/smart-campus-api/api/v1/sensors?type=Temperature"
Expected: 200 OK

16. Try deleting room with sensors attached
curl -i -X DELETE http://localhost:8080/smart-campus-api/api/v1/rooms/LIB-301
Expected: 409 Conflict (Triggered by RoomNotEmptyException)

17. Get readings history for CO2 sensor
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/sensors/CO2-001/readings
Expected: 200 OK (Returns empty list `[]` initially)

18. Add first reading to CO2 sensor
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors/CO2-001/readings \
  -H "Content-Type: application/json" \
  -d '{"id": "R-1", "timestamp": 1700000000, "value": 455.7}'
Expected: 201 Created

19. Get readings history again
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/sensors/CO2-001/readings
Expected: 200 OK (Contains the newly added reading)

20. Check parent sensor updated
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/sensors
Expected: 200 OK (Search the response for "CO2-001" to verify `currentValue` is 455.7. Note: A dedicated GET /{id} for sensors is not implemented per the coursework scope, so we use the collection endpoint).

21. Add second reading to CO2 sensor
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors/CO2-001/readings \
  -H "Content-Type: application/json" \
  -d '{"id": "R-2", "timestamp": 1700000010, "value": 462.3}'
Expected: 201 Created

22. Check sensor updated again
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/sensors
Expected: 200 OK (Search for "CO2-001" to verify `currentValue` is 462.3)

23. Create maintenance sensor
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors \
  -H "Content-Type: application/json" \
  -d '{"id": "TEMP-003", "type": "Temperature", "status": "MAINTENANCE", "currentValue": 21.0, "roomId": "LIB-301"}'
Expected: 201 Created

24. Confirm maintenance sensor status
curl -i -X GET http://localhost:8080/smart-campus-api/api/v1/sensors
Expected: 200 OK (Search for "TEMP-003" to verify status is "MAINTENANCE")

25. Try posting reading to maintenance sensor
curl -i -X POST http://localhost:8080/smart-campus-api/api/v1/sensors/TEMP-003/readings \
  -H "Content-Type: application/json" \
  -d '{"id": "R-3", "timestamp": 1700000020, "value": 23.4}'
Expected: 403 Forbidden (Triggered by SensorUnavailableException)

26. Trigger global 500 error
(Note: Our API strictly enforces the implemented endpoints. The `/debug/error` endpoint was not part of the core coursework specification and is thus not implemented. If you send a request to `/api/v1/debug/error`, it will return a standard `404 Not Found`. To test the `GlobalExceptionMapper` (500 error), you would need to intentionally trigger an unhandled runtime exception inside one of the existing valid endpoints.)
